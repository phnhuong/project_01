## 1. Tổng Quan Dự Án

### 1.1 Mục Đích
Xây dựng một nền tảng quản lý học sinh đơn giản, hiệu quả cho giáo viên chủ nhiệm để quản lý thông tin học sinh, điểm số và xếp hạng trong lớp học.

### 1.2 Phạm Vi
- Quản lý thông tin cá nhân học sinh
- Quản lý điểm kiểm tra và thi
- Tính toán điểm trung bình tự động
- Xếp hạng học sinh theo thành tích

### 1.3 Đối Tượng Người Dùng
- **Chính:** Giáo viên chủ nhiệm
- **Phụ:** Ban giám hiệu, giáo viên bộ môn

### 1.4 Mục Tiêu Kinh Doanh
- Số hóa quy trình quản lý học sinh
- Tiết kiệm 70% thời gian xử lý dữ liệu so với phương pháp thủ công
- Giảm thiểu sai sót trong tính toán điểm số

## 2. Yêu Cầu Chức Năng

### 2.1 Quản Lý Học Sinh

#### 2.1.1 Thêm Học Sinh Mới
**Mô tả:** Cho phép thêm học sinh mới vào hệ thống

**Thông tin cần nhập:**
- Họ và tên (bắt buộc)
- Mã số học sinh (bắt buộc, duy nhất)
- Ngày sinh (bắt buộc)
- Giới tính (Nam/Nữ)
- Địa chỉ
- Số điện thoại phụ huynh
- Email phụ huynh
- Ghi chú

**Validation:**
- Mã số học sinh không được trùng
- Ngày sinh hợp lệ (không vượt quá ngày hiện tại)
- Số điện thoại đúng định dạng (10 số)
- Email đúng định dạng

**Kết quả:**
- Thành công: Hiển thị thông báo, chuyển đến danh sách học sinh
- Thất bại: Hiển thị lỗi cụ thể

#### 2.1.2 Sửa Thông Tin Học Sinh
**Mô tả:** Cập nhật thông tin học sinh đã tồn tại

**Chức năng:**
- Cho phép sửa tất cả các trường thông tin
- Không cho phép sửa mã số học sinh
- Lưu lại lịch sử thay đổi (audit log)

**Validation:** Tương tự phần thêm mới

#### 2.1.3 Xóa Học Sinh
**Mô tả:** Xóa học sinh khỏi hệ thống

**Chức năng:**
- Xác nhận trước khi xóa (confirmation dialog)
- Soft delete: đánh dấu xóa, không xóa vĩnh viễn khỏi database
- Giữ lại dữ liệu điểm số liên quan

**Cảnh báo:**
- "Bạn có chắc chắn muốn xóa học sinh [Tên]? Thao tác này có thể được hoàn tác."

#### 2.1.4 Xem Danh Sách Học Sinh
**Mô tả:** Hiển thị danh sách tất cả học sinh trong lớp

**Chức năng:**
- Hiển thị dạng bảng với các cột: STT, Mã HS, Họ tên, Ngày sinh, Giới tính, Điểm TB, Hạng
- Tìm kiếm theo tên, mã số
- Lọc theo giới tính
- Sắp xếp theo tên, điểm TB, hạng
- Phân trang (20 học sinh/trang)

### 2.2 Quản Lý Điểm Số

#### 2.2.1 Cấu Trúc Điểm Số
**Các loại điểm:**
- **Điểm kiểm tra miệng (KTM):** Hệ số 1
- **Điểm kiểm tra 15 phút (KT15):** Hệ số 1
- **Điểm kiểm tra 1 tiết (KT1T):** Hệ số 2
- **Điểm thi học kỳ (THHK):** Hệ số 3

**Môn học:**
- Toán
- Văn
- Tiếng Anh
- Vật lý
- Hóa học
- Sinh học
- Lịch sử
- Địa lý
- GDCD
- Tin học
- Thể dục

#### 2.2.2 Nhập Điểm
**Mô tả:** Nhập điểm cho học sinh theo môn học

**Giao diện:**
- Chọn môn học (dropdown)
- Chọn loại điểm (dropdown)
- Hiển thị danh sách học sinh dạng bảng
- Nhập điểm trực tiếp vào ô (inline editing)
- Lưu hàng loạt hoặc lưu từng dòng

**Validation:**
- Điểm từ 0 đến 10
- Cho phép 1 chữ số thập phân
- Có thể để trống (chưa có điểm)

**Chức năng bổ sung:**
- Import điểm từ file Excel
- Export danh sách điểm ra Excel

#### 2.2.3 Tính Điểm Trung Bình

**Công thức tính điểm trung bình môn:**
```
ĐTB môn = (Σ(Điểm KTM × 1) + Σ(Điểm KT15 × 1) + Σ(Điểm KT1T × 2) + Σ(Điểm THHK × 3)) / Tổng hệ số
```

**Công thức tính điểm trung bình chung:**
```
ĐTB chung = Σ(ĐTB môn) / Số môn học
```

**Yêu cầu:**
- Tự động tính toán khi có thay đổi điểm
- Làm tròn đến 1 chữ số thập phân
- Hiển thị ngay trên giao diện

#### 2.2.4 Xếp Hạng Học Sinh

**Tiêu chí xếp hạng:**
- Dựa trên điểm trung bình chung
- Sắp xếp từ cao đến thấp
- Hạng 1 = điểm cao nhất

**Xử lý điểm bằng nhau:**
- Cùng hạng nếu điểm trung bình bằng nhau
- Hạng tiếp theo bỏ qua số lượng người cùng hạng
  - Ví dụ: 2 người hạng 1 → người tiếp theo là hạng 3

**Hiển thị:**
- Bảng xếp hạng tổng thể
- Top 10 học sinh xuất sắc
- Biểu đồ phân bố điểm (histogram)

### 2.3 Báo Cáo & Thống Kê

#### 2.3.1 Báo Cáo Cá Nhân
- Thông tin học sinh
- Bảng điểm chi tiết tất cả môn
- Điểm trung bình từng môn
- Điểm trung bình chung
- Xếp hạng
- Export PDF

#### 2.3.2 Báo Cáo Lớp Học
- Tổng số học sinh
- Điểm trung bình lớp
- Số học sinh giỏi/khá/trung bình/yếu
- Tỷ lệ phần trăm mỗi loại
- Biểu đồ thống kê

## 3. Yêu Cầu Phi Chức Năng

### 3.1 Hiệu Năng
- Thời gian tải trang: < 2 giây
- Thời gian xử lý tính toán điểm TB: < 500ms
- Hỗ trợ đồng thời: 50 người dùng

### 3.2 Bảo Mật
- Xác thực người dùng (JWT)
- Mã hóa mật khẩu (bcrypt)
- HTTPS cho mọi request
- SQL injection prevention
- XSS protection
- CSRF protection

### 3.3 Khả Năng Sử Dụng
- Giao diện tiếng Việt
- Responsive design (desktop, tablet)
- Hỗ trợ Chrome, Firefox, Edge (2 phiên bản mới nhất)
- Thông báo lỗi rõ ràng, dễ hiểu

### 3.4 Khả Năng Mở Rộng
- Kiến trúc Monolithic (3-tier architecture)
- API RESTful chuẩn
- Database có thể scale vertical
- Code structure modular để dễ maintain
- Caching layer (Redis - optional cho v2.0)

### 3.5 Backup & Recovery
- Backup database tự động hàng ngày
- Lưu trữ backup 30 ngày
- Khả năng restore trong 1 giờ

### 4.3 Server Configuration Strategy

#### 4.3.1 Process Management (PM2)
- **Backend**: Run as a PM2 process (`pm2 start ecosystem.config.js`).
- **Logs**: Managed by PM2 or written to file system (`/var/log/`).
- **Startup**: Enable PM2 startup hook (`pm2 startup`).

#### 4.3.2 Nginx Configuration
- **Site Config**: Located in `/etc/nginx/sites-available/student-management`.
- **Symlink**: Link to `/etc/nginx/sites-enabled/`.
- **Static Files**: Serve build artifacts from `/var/www/student-management/frontend/dist`.
- **Proxy**: Forward `/api` requests to `http://localhost:3000`.

#### 4.3.3 Database Configuration
- **Service**: PostgreSQL runs as a systemd service (`systemctl start postgresql`).
- **Connection**: Backend connects via `localhost:5432`.
- **Security**: Bind to localhost only, or use restricted pg_hba.conf.

#### 4.3.2 Nginx Configuration (nginx/nginx.conf)
```nginx
server {
    listen 80;
    server_name your-domain.com www.your-domain.com;

    # Frontend (Static Files)
    location / {
        root /home/admin123/qlhs_04/frontend/dist;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # Backend API (Reverse Proxy)
    location /api {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```

### 4.5 Database Schema

#### Table: students
```sql
CREATE TABLE students (
    id SERIAL PRIMARY KEY,
    student_code VARCHAR(20) UNIQUE NOT NULL,
    full_name VARCHAR(100) NOT NULL,
    date_of_birth DATE NOT NULL,
    gender VARCHAR(10),
    address TEXT,
    parent_phone VARCHAR(15),
    parent_email VARCHAR(100),
    notes TEXT,
    is_deleted BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### Table: subjects
```sql
CREATE TABLE subjects (
    id SERIAL PRIMARY KEY,
    subject_name VARCHAR(50) NOT NULL,
    subject_code VARCHAR(10) UNIQUE NOT NULL
);
```

#### Table: scores
```sql
CREATE TABLE scores (
    id SERIAL PRIMARY KEY,
    student_id INTEGER REFERENCES students(id) ON DELETE CASCADE,
    subject_id INTEGER REFERENCES subjects(id),
    score_type VARCHAR(10) NOT NULL, -- 'KTM', 'KT15', 'KT1T', 'THHK'
    score DECIMAL(3,1) CHECK (score >= 0 AND score <= 10),
    semester INTEGER CHECK (semester IN (1, 2)),
    school_year VARCHAR(10),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### Table: users (Authentication)
```sql
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    username VARCHAR(50) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    full_name VARCHAR(100),
    email VARCHAR(100),
    role VARCHAR(20) DEFAULT 'teacher',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## 6. API Endpoints

### 6.1 Student Management

#### GET /api/students
**Mô tả:** Lấy danh sách học sinh  
**Query params:**
- `page`: số trang (default: 1)
- `limit`: số item/trang (default: 20)
- `search`: tìm kiếm theo tên, mã HS
- `gender`: lọc theo giới tính
- `sortBy`: sắp xếp (name, avg_score, rank)

**Response:**
```json
{
  "success": true,
  "data": {
    "students": [...],
    "total": 45,
    "page": 1,
    "totalPages": 3
  }
}
```

#### GET /api/students/:id
**Mô tả:** Lấy chi tiết 1 học sinh

#### POST /api/students
**Mô tả:** Thêm học sinh mới  
**Body:**
```json
{
  "student_code": "HS001",
  "full_name": "Nguyễn Văn A",
  "date_of_birth": "2010-05-15",
  "gender": "Nam",
  "address": "123 Đường ABC, Quận 1, TP.HCM",
  "parent_phone": "0912345678",
  "parent_email": "parent@email.com",
  "notes": ""
}
```

#### PUT /api/students/:id
**Mô tả:** Cập nhật thông tin học sinh

#### DELETE /api/students/:id
**Mô tả:** Xóa học sinh (soft delete)

### 6.2 Score Management

#### GET /api/scores
**Query params:**
- `student_id`: ID học sinh
- `subject_id`: ID môn học
- `semester`: học kỳ

#### POST /api/scores
**Mô tả:** Nhập điểm  
**Body:**
```json
{
  "student_id": 1,
```

#### PUT /api/scores/:id
**Mô tả:** Cập nhật điểm

#### DELETE /api/scores/:id
**Mô tả:** Xóa điểm

### 9.2 Production Environment
**Server Requirements:**
- Ubuntu 22.04 LTS server
- Node.js 20, PostgreSQL 16, Nginx, PM2 installed
- Domain name configured (optional)
- SSL certificates (Let's Encrypt recommended)

**Production Setup Steps:**
1. Clone repository to production server
2. Install dependencies: `npm install` (both frontend & backend)
3. Build frontend: `npm run build`
4. Configure environment variables in `.env` files
5. Setup Database: Create user/db in PostgreSQL
6. Configure Nginx: `/etc/nginx/sites-available/student-management`
7. Start Backend: `pm2 start ecosystem.config.js`
8. Restart Nginx: `systemctl restart nginx`

**SSL Certificate Setup (Let's Encrypt):**
```bash
# Install certbot
sudo apt-get update
sudo apt-get install certbot python3-certbot-nginx

# Generate certificate
sudo certbot --nginx -d your-domain.com
```
